# 문제명
Dreamhack wargame - memory_leakage

## 문제 설명
- 구조체와로컬 버퍼가 스택 상에 연속적으로 배치 됐을 때 널 바이트가 없는 상태에서 문자열을 출력할 때 발생하는 메모리 누수취약점을 다루는 문제입니다. 


## 풀이

### 분석

-문제의 소스 코드를 보면 my_page 구조체와 플래그를 저장하는 flag_buf 배열이 다음과 같이 스택 상에 연속적으로 배치되어 있습니다.

### 취약점

- 이 프로그램의 핵심 취약점은 메뉴 1번에서 이름을 입력받을 때 사용하는 read 함수와 메뉴 2번에서 이름을 출력할 때 사용하는 printf 함수의 특성 차이에서 발생합니다. read 함수는 사용자가 입력한 데이터를 지정한 크기만큼 그대로 가져오지만 문자열의 끝을 나타내는 Null 바이트(\x00)를 맨 뒤에 자동으로 붙여주지 않는 특징이 있습니다. 만약 사용자가 name 배열의 크기인 16바이트를 데이터로 꽉 채워서 입력하게 되면 메모리 상에서 이 이름 데이터의 끝을 알리는 경계가 사라지게 됩니다. 이 상태에서 메뉴 2번을 통해 이름을 출력하면 printf 함수는 Null 바이트를 만날 때까지 스택의 상위 주소를 향해 멈추지 않고 메모리를 계속 읽어 나가게 됩니다. 결과적으로 name 배열의 범위를 넘어 뒤에 붙어 있는 age 변수와 flag_buf 배열의 내용까지 함께 화면에 유출되는 메모리의  취약점이 발생하게 됩니다.

### 익스플로잇

- 메모리가 빈틈없이 연속되어 있다는 특징을 이용하면 복잡한 하드코딩 주소 없이 nc 명령어를 통한 수동 조작만으로도 간단하게 플래그를 알아낼 수 있습니다. 가장 먼저 메뉴 3번을 입력하여 flag_buf에 플래그 파일의 내용을 로드해 둡니다. 그 다음 메뉴 1번을 선택하여 이름 입력란에 16바이트를 완전히 채울 수 있도록 A 문자를 16개 연속으로 입력합니다. 이어서 나이 입력란에는 4바이트 공간을 가득 채우면서도 부호 확장 등에 의해 안전하게 버퍼 간 경계를 지울 수 있는 정수 값인 -1을 입력해 줍니다. 이렇게 하면 name 배열부터 시작해 age 변수를 거쳐 flag_buf 직전까지의 모든 스택 공간이 일반 데이터로 꽉 채워지면서 중간의 Null 바이트가 전부 사라집니다. 마지막으로 메뉴 2번을 선택해 출력을 요청하면 printf 함수가 A로 채워진 이름 영역을 지나 -1이 채워진 나이 영역을 넘어 이미 로드되어 있던 flag_buf 내부의 플래그 문자열까지 널 바이트 없이 쭉 이어져 한 번에 화면에 유출되게 됩니다.

```python

from pwn import *

p = remote('host8.dreamhack.games', 17691)

p.sendlineafter(b'> ', b'3')

p.sendlineafter(b'> ', b'1')

p.sendafter(b'Name: ', b'A' * 16)

p.sendlineafter(b'Age: ', b'-1')

p.sendlineafter(b'> ', b'2')

p.interactive()
```

## 플래그

```
DH{a77ae81944bbbe70adb10d98dc191379}
```

## 배운 점

-이번 문제를 풀면서 이론으로만 접했던 메모리 구조의 연속성과 문자열 처리 함수의 메커니즘이 실제 시스템 해킹에서 어떻게 취약점으로 이어지는지 깊게 배울 수 있었습니다. 특히 read 함수가 입력 데이터의 끝에 Null 바이트(\x00)를 자동으로 처리해주지 않는다는 특성과 printf 함수가 Null 바이트를 만날 때까지 메모리를 연속해서 읽어 나간다는 작동 원리가 결합했을 때 발생하는 Memory Leakage 취약점의 위험성을 명확히 이해하게 되었습니다. 단순한 버퍼 크기 제한뿐만 아니라 문자열의 경계를 확실하게 처리해주는 것이 안전한 코딩의 핵심이라는 점을 깨달았습니다.

또한 처음에는 로컬 환경의 gdb 디버깅 주소와 원격 서버의 주소가 달라 구조체 변수를 직접 타겟팅하는 exploit 코드를 구성하는 데 어려움을 겪기도 했지만 복잡한 하드코딩 주소에 의존하는 대신에 스택 상에서 버퍼들이 빈틈없이 이어져 있다는 구조적 특징을 활용하여 경계를 채워버리는 직관적인 시나리오로 문제를 해결하면서 시스템 구조를 넓은 시야에서 바라보는 안목을 기를 수 있었습니다. 앞으로 코드를 작성하거나 취약점을 분석할 때 프로그램이 메모리 상에서 어떻게 배치되고 제어되는지 데이터를 밀도 있게 관찰해야겠다고 느꼈습니다.
