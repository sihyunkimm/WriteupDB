# BasicIncome_Crackme

## 문제 설명
> BasicIncome eligibility calculator — household_id를 입력하면 Eligible 여부와 Voucher를 출력한다.

## 풀이

### 분석

정적 링크된 stripped ELF 64-bit 바이너리. 실행하면 `household_id>` 프롬프트를 출력하고 입력을 받는다.

Ghidra로 디컴파일하면 메인 로직(`FUN_00401680`)에서 두 단계 연산이 보인다.

**1단계 — 입력 검증 (16 bytes)**

```c
if ((&DAT_0047e0d0)[i] != ((&DAT_0047e0e0)[i & 7] ^ input[i])) {
    puts("Ineligible.");
    return;
}
```

입력의 각 바이트가 `DAT_0047e0d0[i] ^ DAT_0047e0e0[i & 7]` 와 일치해야 통과.

**2단계 — Voucher(플래그) 생성 (40 bytes)**

```c
output[i] = input[i & 0xf] ^ (&DAT_0047e0a0)[i];
```

검증 통과한 입력으로 XOR하여 Voucher를 출력한다.

### 풀이 과정

1. pwntools ELF로 바이너리에서 세 DAT 주소의 bytes를 읽는다.
2. 검증 XOR을 역산해 올바른 16-byte 입력을 복원한다.
3. 복원된 입력으로 Voucher XOR 연산을 수행해 플래그를 얻는다.

```python
from pwn import ELF, context
context.log_level = 'error'

elf = ELF('./BasicIncome_Crackme', checksec=False)
dat_cmp = elf.read(0x47e0d0, 16)   # 비교 대상
dat_key = elf.read(0x47e0e0, 8)    # XOR 키 (8-byte 반복)
dat_out = elf.read(0x47e0a0, 40)   # Voucher XOR 테이블

input_bytes = bytes(dat_cmp[i] ^ dat_key[i % 8] for i in range(16))
voucher     = bytes(input_bytes[i % 16] ^ dat_out[i] for i in range(40))
print(voucher.decode('latin-1'))
```

## 플래그

```
SDG{5c7508791181c684f0ea38e326d6f584}
```

## 배운 점

- Ghidra 디컴파일 결과에서 `(&DAT_X)[i]` 패턴은 배열 접근이며 pwntools `elf.read(addr, n)`으로 직접 읽을 수 있다.
- 2단계 XOR crackme는 검증 역산 → 플래그 생성 두 단계로 분리해 풀면 된다.
