# 문제명
old Sessions

## 문제 설명

> Description
Proper session timeout controls are critical for securing user accounts. If a user logs in on a public or shared computer but doesn’t explicitly log out (instead simply closing the browser tab), and session expiration dates are misconfigured, the session may remain active indefinitely.
This then allows an attacker using the same browser later to access the user’s account without needing credentials, exploiting the fact that sessions never expire and remain authenticated.
Additional details will be available after launching your challenge instance

- 문제 URL / 파일 등 접속 정보
http://dolphin-cove.picoctf.net:63096/
## 풀이
이 문제는 세션 관리(Session Management)의 취약점을 이용하는 웹 해킹 문제이다.
문제 설명에서 “세션이 만료되지 않는다”는 점과 /sessions 페이지 힌트를 통해 세션 관련 취약점이 존재함을 알 수 있다.



하지만 이 문제에서는 다음과 같은 취약점이 존재한다:

1. /sessions 페이지에서 **모든 사용자 세션이 노출됨
2. 세션 값만으로 사용자 인증이 이루어짐
3. 추가적인 검증 없이 쿠키 값만 바꾸면 다른 사용자로 로그인 가능


 계정을 생성하거나 로그인
 브라우저 개발자 도구(F12) → Application → Cookies에서 session 값 확인

문제 힌트에 따라 /sessions페이지에 접근:
mary_jones_8992
2024-2-20 14:50
Hey I found a strange page at /sessions 이게 엄청 큰 힌트. 지금 페이지 뒤에 이거 붙여서 검색하니
1) session:w8250yPBZUE3WYDL8PDJ9mECUIfXbkiglBXYJ0kAp6Y, {'_permanent': True, 'key': 'admin'}

2) session:590R1QUFBWcDe0K_9nmrDhCYN04N70keGHuHBEfbrcQ, {'_permanent': True, 'key': 'jsh'} 이런거 나왔음.


여기서 다음과 같은 정보를 확인할 수 있다:


session:w8250yPBZUE3WYDL8PDJ9mECUIfXbkiglBXYJ0kAp6Y, {'key': 'admin'}
session:590R1QUFBWcDe0K_9nmrDhCYN04N70keGHuHBEfbrcQ, {'key': 'jsh'}

admin 계정의 session 값 획득


세션 하이재킹 (Session Hijacking)
개발자 도구 → Application → Cookies 이동
기존 `session` 값을 아래 값으로 변경:


w8250yPBZUE3WYDL8PDJ9mECUIfXbkiglBXYJ0kAp6Y


페이지 새로고침(F5)
admin 계정으로 로그인된 상태가 됨
관리자 페이지 또는 메인 페이지에서 flag 확인

picoCTF{s3t_s3ss10n_3xp1rat10n5_77b6684a}



세션 값을 단순히 변경하는 것만으로 admin계정으로 인증 우회가 가능했다.



이 문제에서 사용된 취약점은 다음과 같다:

Session Exposure (세션 정보 노출)
Improper Session Management (잘못된 세션 관리)
Session Hijacking (세션 탈취)



이 문제는 세션 관리의 중요성을 보여주는 문제로, 단순한 쿠키 값 조작만으로도 인증을 우회할 수 있음을 확인할 수 있었다.
