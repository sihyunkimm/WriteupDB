# 문제명

## 문제 설명

> Let's go hunt down some treasure! The flag is split into 4 parts. I'll give you the first one right here: tjctf

- https://treasure-hunt.tjc.tf 정보

## 풀이

### 분석
처음 문제 페이지에 접속하면 해적과 관련된 간단한 페이지가 나타난다. 페이지 자체에서 입력값을 받는 부분은 없었다. 페이지 소스는 다음과 같다.
```html
<h1>Learn about pirates!</h1>
<h2>Wow!</h2>
<form method="POST">
    <input type="submit" value="Learn More">
</form>
<img src="/static/ship.png" alt="ship">
<p hidden>_and_</p>
```
여기서 flag 조각 _and_를 발견하였다.

### 취약점

이 문제는 특정 취약점이 존재한다기보다는, 웹 서비스에 노출된 여러 정보를 순서대로 확인하는 정보 수집형 문제이다.
HTML 소스에 숨겨진 태그, POST 요청에 대한 HTTP 응답 헤더, Set-Cookie에 포함된 값, robots.txt에 노출된 숨겨진 경로 등이 예시이다.
특히 Set-Cookie 헤더에 플래그 조각이 들어 있었기 때문에, 브라우저 화면에 보이는 본문만 확인해서는 플래그를 구할 수 없다.

### 익스플로잇
1. 문제 설명에서 첫 번째 조각 `tjctf`를 확인한다.
2. 메인 페이지 HTML 소스를 확인하여 조각 `_and_`를 얻는다.
    curl.exe -i https://treasure-hunt.tjc.tf/
3. 메인 페이지의 form이 POST 방식이므로 POST 요청을 보내고 응답 헤더에서 세 번째 조각 `s1lv3r`를 얻는다.
    curl.exe -i -X POST https://treasure-hunt.tjc.tf/
    응답헤더: Set-Cookie: silver_coffer=`{s1lv3r`; Path=/

4. 숨겨진 경로를 찾기 위해 robots.txt를 확인한다.
    curl.exe -i https://treasure-hunt.tjc.tf/robots.txt
5. 해당 경로에서 Disallow: /gold-coffer 를 확인 후 해당 경로에 접근한다.
    curl.exe -i https://treasure-hunt.tjc.tf/gold-coffer 
    이후 마지막 조각 `g0ld}` 확인
6. 획득한 플래그 조각들 조합하여 플래그 완성
    tjctf{s1lv3r_and_g0ld}
