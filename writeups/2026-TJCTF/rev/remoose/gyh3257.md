# remoose

## 문제 설명
> Edible-nest Swiftlet  
> Aerodramus fuciphagus  
> Documented by hassaanm.

바이너리 파일 `chall`이 제공된다.

## 풀이

### 분석

`file chall`을 실행하면 `data`로 인식된다.

```
$ xxd chall | head -4
00000000: 7f45 4c4b 0201 0120 2020 2020 2020 2020  .ELK...
```

정상 ELF 매직은 `7f 45 4c 46` (`\x7fELF`)이지만, 이 파일은 `7f 45 4c 4b` (`\x7fELK`)이다.  
**ELK** = Moose(말코손바닥사슴)의 일종. 문제 이름 "remoose"와 연결된다.

바이트 분포를 분석하면:
- `0x20` (space): 전체의 **83.9%** 차지

일반 ELF 바이너리에서는 `0x00` (null)이 가장 많아야 한다. 따라서 변환 규칙:
- `0x00` → `0x20` (null → space)
- magic byte: `F` → `K` (ELF → ELK)

### 풀이 과정

**1단계: ELF 복원**

```python
with open('chall', 'rb') as f:
    data = bytearray(f.read())

# 모든 0x20을 0x00으로 복원
data = bytearray(b if b != 0x20 else 0x00 for b in data)

# magic byte 수정: K -> F
data[3] = 0x46

with open('chall_fixed', 'wb') as f:
    f.write(data)
```

```
$ file chall_fixed
chall_fixed: ELF 64-bit LSB pie executable, x86-64, not stripped
```

**2단계: 심볼 분석**

```
$ readelf -s chall_fixed | grep -E "flag|main"
flag2  0x1229  FUNC
flag4  0x11ee  FUNC
flag3  0x115a  FUNC
flag1  0x11c9  FUNC
main   0x1145  FUNC
flag   0x117f  FUNC
```

`flag1`~`flag4` 및 `flag` 함수가 존재한다.

**3단계: 함수 추적**

각 함수를 디스어셈블하면 `putchar()`와 `printf()`로 문자를 출력하고 다음 함수를 호출하는 구조:

| 함수 | 출력 | 다음 호출 |
|------|------|-----------|
| `flag` | `t`, `j`, `c`, `t`, `f{` | `flag1` |
| `flag1` | `5`, `m` | `flag2` |
| `flag2` | `a`, `1`, `1`, `_` | `flag3` |
| `flag3` | `m`, `0` | `flag4` |
| `flag4` | `0`, `s`, `3`, `}` | - |

호출 체인: `main` → `flag` → `flag1` → `flag2` → `flag3` → `flag4`

출력을 이어 붙이면:

```
tjct + f{ + 5m + a11_ + m0 + 0s3 + }
= tjctf{5ma11_m00s3}
```

**"5ma11_m00s3"** = l33t speak for **"small_moose"** → 문제 이름 **"remoose"**!

## 플래그

```
tjctf{5ma11_m00s3}
```

## 배운 점

- ELF magic byte 변조 (`\x7fELF` → `\x7fELK`) + null byte를 space(0x20)로 치환하는 간단한 난독화 기법
- 바이트 분포 분석으로 변환 패턴을 빠르게 파악할 수 있음
- 함수 분할 + 재귀적 call chain으로 flag를 분산 출력하는 패턴
